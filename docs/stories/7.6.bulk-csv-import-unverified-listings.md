# Story 7.6: Bulk CSV Import for Unverified Listings - Brownfield Addition

## Status
Ready for Review

## Story
**As an** admin managing crew profiles,
**I want** to upload a CSV file with bulk crew data to create multiple unverified listings at once,
**so that** I can efficiently populate the platform with crew profiles that can later be reviewed and verified.

## Story Context

**Existing System Integration:**
- Integrates with: Admin dashboard (`apps/web/app/(auth)/admin/page.tsx`), AdminDashboardTabs component (`apps/web/components/admin/AdminDashboardTabs.tsx`), profile creation API (`apps/web/app/api/admin/profiles/route.ts`), ProfileService (`apps/web/lib/services/profileService.ts`)
- Technology: Next.js App Router, React, TypeScript, Supabase, FormData for file uploads
- Follows pattern: File upload pattern from photo upload API (`apps/web/app/api/admin/profiles/[id]/photo/route.ts`), profile creation pattern from existing POST endpoint, admin authentication middleware
- Touch points: Admin dashboard UI, bulk import API endpoint, ProfileService.createPendingProfile() method, CSV parsing utility

## Acceptance Criteria

**Functional Requirements:**

1. Admin dashboard has a new "Bulk Import" tab or section visible alongside "Pending Review" and "All Profiles" tabs
2. Bulk import interface includes a file input that accepts CSV files only
3. File input shows validation error if non-CSV file is selected
4. CSV file is parsed and validated for required columns: name, primary_role, primary_location_city, primary_location_state, contact_email
5. CSV parsing handles optional columns: contact_phone, bio, portfolio_url, website, instagram_url, vimeo_url, union_status, years_experience, secondary_roles (comma-separated), specialties (comma-separated), additional_markets (format TBD)
6. Each valid CSV row creates a profile with `profile_status: 'pending_review'` and `is_verified: false`
7. Profiles created via bulk import have `is_claimed: false` (unclaimed)
8. Slug generation works for bulk-imported profiles (using existing slug generation service)
9. Import process shows progress feedback (e.g., "Processing row 5 of 100...")
10. Import results display summary: total rows, successful imports, failed imports with error details
11. Failed rows show specific error messages (e.g., "Row 3: Missing required field 'contact_email'")
12. Successfully imported profiles appear in "Pending Review" tab
13. CSV file size limit enforced (e.g., max 10MB or 1000 rows, configurable)

**Integration Requirements:**

14. Bulk import uses existing `requireAdmin()` middleware for authentication
15. Bulk import uses existing `ProfileService.createPendingProfile()` method for profile creation
16. Bulk import follows existing profile validation rules
17. Existing profile creation API (`POST /api/admin/profiles`) continues to work unchanged
18. Existing admin dashboard tabs and functionality remain unchanged
19. Bulk-imported profiles follow same slug generation and uniqueness logic as manually created profiles
20. No breaking changes to Profile type or database schema

**Quality Requirements:**

21. CSV parsing handles edge cases: quoted fields, commas within fields, empty rows, trailing whitespace
22. Error handling is graceful - one failed row doesn't stop entire import
23. Import process is non-blocking (shows progress, doesn't freeze UI)
24. File validation happens before parsing (file type, size)
25. Appropriate error messages displayed for validation failures
26. Code follows existing patterns and coding standards
27. TypeScript types are properly defined for CSV row structure

## Technical Notes

- **Integration Approach:**
  - Add new "Bulk Import" tab to `AdminDashboardTabs.tsx` component
  - Create new component `BulkImportSection.tsx` for CSV upload UI
  - Create new API endpoint `POST /api/admin/profiles/bulk-import` that:
    - Accepts multipart/form-data with CSV file
    - Validates file type (CSV only) and size
    - Parses CSV using a CSV parsing library (e.g., `papaparse` or native parsing)
    - Validates each row against profile creation schema
    - Creates profiles using `ProfileService.createPendingProfile()` in batch
    - Returns import results with success/failure counts and error details
  - Use existing `requireAdmin()` middleware for authentication
  - Use existing `ProfileService.createPendingProfile()` method (sets `profile_status: 'pending_review'`, `is_verified: false`)
  - Handle CSV parsing errors gracefully (skip invalid rows, continue processing)
  - Show progress feedback during import (could use streaming or batch processing with progress updates)

- **Existing Pattern Reference:**
  - File upload pattern: `apps/web/app/api/admin/profiles/[id]/photo/route.ts` - shows FormData handling, file validation
  - Profile creation: `apps/web/app/api/admin/profiles/route.ts` - shows validation schema, profile service usage
  - Admin authentication: `apps/web/lib/middleware/auth.ts` - `requireAdmin()` function
  - Profile service: `apps/web/lib/services/profileService.ts` - `createPendingProfile()` method
  - Admin dashboard tabs: `apps/web/components/admin/AdminDashboardTabs.tsx` - tab structure pattern

- **Key Constraints:**
  - Must maintain backward compatibility with existing profile creation flow
  - CSV parsing must handle various CSV formats (quoted fields, escaped commas, etc.)
  - Import should be performant for large files (consider batching or streaming)
  - Error messages must be clear and actionable for admins
  - Must not create duplicate profiles (slug uniqueness already handled by service)
  - File size limits to prevent memory issues with very large CSV files

## Risk and Compatibility Check

**Minimal Risk Assessment:**

- **Primary Risk:** Large CSV files causing memory issues or timeouts, invalid CSV format breaking parser, duplicate email addresses creating conflicts
- **Mitigation:** 
  - Enforce file size limits (e.g., max 10MB or 1000 rows)
  - Use robust CSV parsing library that handles edge cases
  - Process rows in batches to avoid memory issues
  - Validate email uniqueness before creating profiles (or handle gracefully)
  - Set reasonable timeout limits for import process
- **Rollback:** Remove bulk import tab/component and API endpoint; existing profiles remain unaffected

**Compatibility Verification:**

- [x] No breaking changes to existing APIs (new endpoint only)
- [x] Database changes (none required - uses existing schema)
- [x] UI changes follow existing design patterns (admin dashboard tabs)
- [x] Performance impact is minimal (import is admin-only, infrequent operation)
