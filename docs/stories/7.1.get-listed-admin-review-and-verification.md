# Story 7.1: Get Listed Admin Review & Verified Subscription System

## Status
Done

## Story
**As a** platform owner,
**I want** crew members to submit "Get Listed" requests that require admin review and approval, with an optional paid verification tier ($5/month) that includes ID verification and search priority,
**so that** I can maintain quality control, prevent spam, and monetize verified profiles while keeping basic listings free.

## Acceptance Criteria
1. "Get Listed" flow creates profile in "pending_review" status (not immediately live)
2. Admin receives email notification when new "Get Listed" request is submitted
3. Email includes profile details and optional ID photo attachment
4. Admin can approve/reject profiles from admin dashboard
5. Approved profiles go live with "Listed" status (free, unverified)
6. Users can claim their approved profile (email verification) to edit it
7. Claimed users can upgrade to "Verified" status ($5/month subscription)
8. Verified upgrade requires ID photo upload
9. Verified profiles receive priority in search results (appear before unverified)
10. Verified profiles display verification badge on profile page
11. Subscription management (cancel, renew) for verified users
12. Search results sort verified profiles first, then unverified (by relevance)
13. All code follows coding standards and project structure

### Database Schema

**Profiles table additions:**
```sql
ALTER TABLE public.profiles
  ADD COLUMN profile_status TEXT CHECK (profile_status IN ('pending_review', 'approved', 'rejected')) DEFAULT 'pending_review',
  ADD COLUMN is_verified BOOLEAN DEFAULT FALSE,
  ADD COLUMN verification_id_url TEXT,
  ADD COLUMN verification_requested_at TIMESTAMPTZ,
  ADD COLUMN verified_at TIMESTAMPTZ;

CREATE INDEX idx_profiles_status ON public.profiles(profile_status);
CREATE INDEX idx_profiles_verified ON public.profiles(is_verified) WHERE is_verified = TRUE;
```

**New subscriptions table:**
```sql
CREATE TABLE IF NOT EXISTS public.subscriptions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES public.users(id) ON DELETE CASCADE,
    profile_id UUID NOT NULL REFERENCES public.profiles(id) ON DELETE CASCADE,
    stripe_subscription_id TEXT UNIQUE NOT NULL,
    status TEXT NOT NULL CHECK (status IN ('active', 'canceled', 'past_due', 'unpaid')),
    current_period_start TIMESTAMPTZ NOT NULL,
    current_period_end TIMESTAMPTZ NOT NULL,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_subscriptions_user_id ON public.subscriptions(user_id);
CREATE INDEX idx_subscriptions_profile_id ON public.subscriptions(profile_id);
CREATE INDEX idx_subscriptions_stripe_id ON public.subscriptions(stripe_subscription_id);
```

### API Specifications

**POST `/api/profiles/get-listed`**
- Authentication: None (public endpoint)
- Request body: Profile data + optional ID photo
- Response: 201 Created with profile ID
- Creates profile with `profile_status: 'pending_review'`

**POST `/api/admin/profiles/[id]/approve`**
- Authentication: Required (admin role)
- Response: 200 OK
- Sets `profile_status: 'approved'`, profile goes live
- Sends approval email to user

**POST `/api/admin/profiles/[id]/reject`**
- Authentication: Required (admin role)
- Request body: `{ reason?: string }`
- Response: 200 OK
- Sets `profile_status: 'rejected'`
- Sends rejection email to user

**POST `/api/subscriptions/create-checkout-session`**
- Authentication: Required (user must be logged in, profile must be claimed)
- Response: 200 OK with Stripe Checkout session URL
- Creates Stripe Checkout session for $5/month subscription

**POST `/api/webhooks/stripe`**
- Authentication: Stripe webhook signature verification
- Handles: subscription.created, subscription.updated, subscription.deleted, invoice.payment_succeeded, invoice.payment_failed
- Updates subscription status in database

**POST `/api/subscriptions/cancel`**
- Authentication: Required (user must own subscription)
- Response: 200 OK
- Cancels Stripe subscription (cancels at period end)

**POST `/api/subscriptions/reactivate`**
- Authentication: Required (user must own subscription)
- Response: 200 OK
- Reactivates canceled Stripe subscription
