# Story 7.4: Profile Header UI Redesign - Enhanced Visual Hierarchy

## Status
Ready for Review

## Story
**As a** production coordinator viewing crew profiles,
**I want** to see a more visually prominent and well-organized profile header with larger photo and clearer information hierarchy,
**so that** I can quickly assess crew members' professional identity and credentials at a glance.

## Story Context

**Existing System Integration:**
- Integrates with: Profile page (`apps/web/app/(public)/crew/[slug]/page.tsx`), ProfileHeader component (`apps/web/components/profile/ProfileHeader.tsx`), Avatar component (`apps/web/components/ui/Avatar.tsx`)
- Technology: Next.js App Router, React, Tailwind CSS, shadcn/ui components, TypeScript
- Follows pattern: Existing ProfileHeader component structure, Tailwind utility classes, shadcn/ui design system
- Touch points: ProfileHeader component, Avatar component sizing, ProfileStatusBadge component, page layout container

## Acceptance Criteria

**Functional Requirements:**

1. Profile photo is increased from current ~120px to 160px on desktop (maintaining responsive sizing)
2. Profile photo has 4px white border with subtle shadow (0 2px 8px rgba(0,0,0,0.1))
3. Profile photo remains circular and properly positioned
4. Name font size increased to 32px (from current ~24px) with weight 700
5. Title/role font size increased to 20px (from current ~18px) with weight 500
6. Location font size set to 16px with proper icon spacing (8px between icon and text)
7. Status badge is positioned inline next to name (not separate line)
8. Experience badge displays below location with format: "👤 Pronouns • 🕐 X years experience" (if available)
9. Overall spacing between header elements increased (16-32px gaps instead of 8-12px)
10. Header section has improved visual hierarchy with proper typography scale
11. All existing functionality continues to work (contact reveal, claim button, links, etc.)

**Integration Requirements:**

12. New styling follows existing Tailwind CSS patterns and utility classes
13. Component structure remains compatible with existing ProfileHeader props interface
14. Responsive breakpoints maintained (mobile, tablet, desktop)
15. Avatar component works correctly with new sizing requirements (may need new size variant or custom sizing)
16. ProfileStatusBadge component positioning updated to inline with name
17. No breaking changes to Profile type or data structure

**Quality Requirements:**

18. Visual design matches specification (font sizes, spacing, colors within design system)
19. Responsive design works on mobile (< 640px), tablet (640px-1024px), and desktop (> 1024px)
20. Accessibility maintained (proper heading hierarchy, alt text, keyboard navigation)
21. No layout shift or visual regressions in existing functionality
22. Code follows existing coding standards and component patterns

## Technical Notes

- **Integration Approach:** 
  - Update `ProfileHeader.tsx` component with new Tailwind classes for typography, spacing, and layout
  - Modify Avatar component usage to support 160px size (either extend size prop or use custom className override)
  - Update ProfileStatusBadge positioning to be inline with h1 name element
  - Adjust container spacing and padding to match new design specifications
  - Add experience/pronouns display section if data is available in Profile type
  - Maintain all existing conditional rendering logic (bio, contact, links, badges, claim button)

- **Existing Pattern Reference:**
  - ProfileHeader component: `apps/web/components/profile/ProfileHeader.tsx` - current implementation
  - Avatar component: `apps/web/components/ui/Avatar.tsx` - supports size prop (sm, md, lg, xl)
  - ProfileStatusBadge: `apps/web/components/profile/ProfileStatusBadge.tsx` - badge component
  - Tailwind design tokens: Check existing color/spacing system in tailwind.config
  - Responsive patterns: Existing md: breakpoints in ProfileHeader

- **Key Constraints:**
  - Must maintain backward compatibility with Profile type (no schema changes)
  - Avatar component xl size is 200px, may need custom sizing for 160px or extend component
  - Must preserve all existing functionality (contact reveal, claim modal, external links)
  - Responsive design must not break on mobile devices
  - Typography changes should use existing Tailwind font scale or extend if needed

- **Mitigation:** Use custom className override for Avatar or extend Avatar component with new size variant; test responsive breakpoints thoroughly
- **Rollback:** Revert ProfileHeader.tsx changes to previous version

**Compatibility Verification:**

- [x] No breaking changes to existing APIs (UI-only change)
- [x] Database changes (none required)
- [x] UI changes follow existing design patterns (Tailwind, shadcn/ui)
- [x] Performance impact is negligible (CSS changes only, no new data fetching)
