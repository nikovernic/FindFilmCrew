# Story 7.5: Get Listed Form Enhancements - Bio Character Limit & Specialty/Genre Field

## Status
Pending

## Story
**As a** crew member submitting my profile,
**I want** to provide a more detailed bio (up to 2000 characters) with helpful guidance, and specify my specialties/genres (documentary, commercial, feature film, food, sports, fashion, etc.),
**so that** I can better showcase my experience and help producers find me for the right types of projects.

## Story Context

**Existing System Integration:**
- Integrates with: Get Listed form component (`GetListedForm.tsx`) and profile creation API
- Technology: Next.js, React Hook Form, Zod validation, Supabase database
- Follows pattern: Multi-select field pattern (similar to `secondary_roles` implementation)
- Touch points: Database schema, form validation, API endpoint, TypeScript types

## Acceptance Criteria

**Functional Requirements:**

1. Bio field character limit increased from 250 to 2000 characters
2. Bio field label updated to: "Bio (Optional, max 2000 characters)"
3. Bio field placeholder/helper text updated to: "Bios help crew book more gigs! Get creative, tell us what your specialty is, your experience, and what you're like on set!"
4. New `specialties` field added to profiles table as TEXT[] array (similar to `secondary_roles`)
5. Specialty/genre field added to Get Listed form with multi-select functionality
6. Specialty options include: Documentary, Commercial, Feature Film, Music Video, TV, Corporate, Food, Sports, Fashion, Reality TV, Live Events, Web Content, Branded Content, Short Film, Other
7. Users can select multiple specialties
8. Specialty field is optional
9. Specialties display on profile pages (if provided)
10. Database constraint updated: `bio TEXT CHECK (char_length(bio) <= 2000)`

**Integration Requirements:**

11. Existing profile creation API continues to work unchanged
12. Existing profile display pages continue to work unchanged
13. New functionality follows existing multi-select pattern (like `secondary_roles`)
14. Database migration is backward compatible (existing profiles work without specialties)

**Quality Requirements:**

15. Change is covered by appropriate tests
16. TypeScript types updated to include `specialties` field
17. No regression in existing functionality verified
18. Form validation properly handles new field
